  <name>Create GitHub Actions E2E workflow with caching</name>
  <files>.github/workflows/e2e-tests.yml</files>
  <action>
    Create .github/workflows/e2e-tests.yml with:

    Trigger:
    - on: pull_request (branches: [main])
    - on: push (branches: [main])
    - on: schedule (cron: '0 2 * * *' - nightly at 2 AM UTC)
    Per CONTEXT.md: PR + Scheduled (nightly)

    Job:
    - runs-on: ubuntu-latest
    - steps:
      1. Checkout repository: uses: actions/checkout@v4
      2. Setup Node.js: uses: actions/setup-node@v4 (node-version: '20')
      3. Set up Docker Buildx: uses: docker/setup-buildx-action@v3
      4. Cache Docker layers: uses: actions/cache@v3
         - path: /tmp/.buildx-cache
         - key: ${{ runner.os }}-buildx-${{ github.sha }}
         - restore-keys: ${{ runner.os }}-buildx-
         Per CONTEXT.md: Docker layer caching for speedup
      5. Build Docker image: docker build -f tests/e2e/Dockerfile -t opencode-carl:e2e --cache-from type=local,src=/tmp/.buildx-cache --cache-to type=local,dest=/tmp/.buildx-cache-new,mode=max .
      6. Run E2E tests: run: npm run test:e2e:ci
         Per CONTEXT.md: npm run test:e2e:ci script (created in next task)
    - Retry logic: Use continue-on-error or custom retry step
      Per CONTEXT.md: Retry once, then block PR
      Implementation: Wrap test execution in bash loop with one retry
    - Artifact collection (on failure): actions/upload-artifact@v4
      - Upload: Docker container logs (docker logs carl-e2e)
      - Upload: .carl/ directory state (tarball)
      - Upload: Session files content
      - Upload: OpenCode stdout/stderr
      Retention: 14 days for failure artifacts
      Per CONTEXT.md: All of the following artifacts on failure

    Per CONTEXT.md: Skip draft PRs (copy pattern from test.yml if needed)
    Per CONTEXT.md: Concurrency group to cancel outdated runs
  </action>
  <verify>workflow YAML validates: yamllint .github/workflows/e2e-tests.yml or GitHub Actions UI shows no syntax errors</verify>
