<task type="auto">
  <name>Task 2: Add strict-but-nonfatal manifest and domain parsing</name>
  <files>src/carl/validate.ts, src/carl/loader.ts</files>
  <action>
Implement parsing and validation logic aligned with `hooks/carl-hook.py` and phase decisions:

- Create `src/carl/validate.ts` with helpers to parse manifest lines and domain files. Use Zod where helpful, but keep behavior nonfatal:
  - Unknown manifest keys: warn and ignore.
  - Malformed manifest lines: skip line with warning.
  - Active domain with missing file: warn and skip that domain.
  - Invalid rule keys inside a domain: warn and skip the invalid entries, keep valid ones.
  - Ignore comments and blank lines in domain files.
  - Domain filenames must be lowercase; warn and skip non-lowercase files.
  - Only domains referenced in the manifest are loaded; extra files are ignored.

- Support manifest fields: `{DOMAIN}_STATE`, `{DOMAIN}_RECALL`, `{DOMAIN}_EXCLUDE`, `{DOMAIN}_ALWAYS_ON` (plus `DEVMODE` and `GLOBAL_EXCLUDE` as known keys for parity).
- For CONTEXT domains, accept bracket rule keys (`FRESH_RULES`, `FRESH_RULE_1`, `MODERATE_RULE_1`, etc.) so those lines are not treated as invalid.

Wire `src/carl/loader.ts` to use the validator outputs to produce a final discovery result with collected warnings.
  </action>
  <verify>node -e "const fs=require('fs'); console.log(fs.readFileSync('src/carl/validate.ts','utf8').includes('warn'));"</verify>
  <done>Invalid manifest or domain entries are reported as warnings and do not abort discovery.</done>
