  <action>
Add TypeScript scaffolding for the OpenCode plugin and CARL rule loader interface:

1) `src/carl/types.ts`: define shared types for discovery results (sources, domains, warnings) that the loader will return.
2) `src/carl/loader.ts`: export `loadCarlRules()` that returns an empty-but-shaped discovery result (sources empty, warnings empty) so the plugin can call it before real parsing is implemented.
3) `.opencode/plugins/carl.ts`: implement the OpenCode plugin entrypoint using the OpenCode plugin API (per docs). Export default plugin (e.g., `definePlugin`) and in the startup hook call `loadCarlRules()` and log a concise summary + any warnings. Keep logging format stable but simple (use `console.warn` for warnings). During startup, detect whether both `.opencode/plugins/carl.ts` and `~/.config/opencode/plugins/carl.ts` exist and emit a warning about duplicate plugin placement.

Do not add rule discovery logic yet; this plan is only establishing the entrypoint and interface.
  </action>
  <verify>node -e "const fs=require('fs'); const ok=fs.readFileSync('.opencode/plugins/carl.ts','utf8').includes('loadCarlRules'); console.log(ok);"</verify>
  <done>Plugin entrypoint compiles against the loader interface and logs a discovery summary on startup.</done>
</task>
