<task type="auto">
  <name>Create duplicate-detector.ts module</name>
  <files>src/carl/duplicate-detector.ts</files>
  <action>
    Create `src/carl/duplicate-detector.ts` with the following exports:

    1. `registerPluginLoad(pluginPath: string): void`
       - Normalizes path using `path.resolve()` and replaces backslashes
       - Adds normalized path to global `loadedPluginPaths` Set
       - Called once per plugin instance at initialization

    2. `checkDuplicateLoad(currentPath: string, sessionId: string): DuplicateCheckResult`
       - Normalizes current path for comparison
       - Checks if any existing path differs from current
       - Uses session-scoped `duplicateWarningGuard` Set to warn once per session
       - Returns `{ isDuplicate: boolean; existingPath: string | null; warningEmitted: boolean }`

    3. `getDuplicateWarning(currentPath: string, existingPath: string): string`
       - Returns formatted warning message with `[carl]` prefix
       - Includes both paths for user visibility
       - Provides actionable guidance: "Remove one from your opencode.json plugins array"
       - Format:
         ```
         [carl] Duplicate plugin detected
         
         Multiple CARL plugins are loaded:
           - {existingPath}
           - {currentPath}
         
         This may cause unexpected behavior. Remove one from your opencode.json plugins array.
         ```

    4. `clearDuplicateState(): void`
       - Clears both `loadedPluginPaths` and `duplicateWarningGuard` Sets
       - For testing purposes only

    Internal state (module-level):
    ```typescript
    const loadedPluginPaths = new Set<string>();
    const duplicateWarningGuard = new Set<string>();
    ```

    Types:
    ```typescript
    interface DuplicateCheckResult {
      isDuplicate: boolean;
      existingPath: string | null;
      warningEmitted: boolean;
    }
    ```

    Path normalization:
    ```typescript
    const normalized = path.resolve(pluginPath).replace(/\\/g, '/');
    ```

    DO NOT:
    - Do not warn on every turn - use session guard
    - Do not fail plugin load on duplicate - first load wins
    - Do not compare paths without normalization (symlinks, case differences)
  </action>
  <verify>
    ```bash
    # Verify file exists with expected exports
    grep -E "(registerPluginLoad|checkDuplicateLoad|getDuplicateWarning)" src/carl/duplicate-detector.ts
    ```
  </verify>
  <done>
    `src/carl/duplicate-detector.ts` exists with all exported functions. Session-scoped warning guard prevents repeated warnings. Path normalization handles cross-platform differences.
  </done>

       - Resolve to get plugin directory:
       ```typescript

         ? __dirname
         : path.dirname(fileURLToPath(import.meta.url));

       ```

    3. Add `import { fileURLToPath } from "url"` if using ESM path resolution

    4. In `experimental.chat.system.transform` handler, after setup check, add duplicate check:

       ```typescript
         const warning = getDuplicateWarning(PLUGIN_PATH, duplicateCheck.existingPath);
         console.warn(warning);
       }
       ```

    Placement: After setup check, before rule loading.

    DO NOT:

    - Do not check duplicate on every turn (session guard handles this)
