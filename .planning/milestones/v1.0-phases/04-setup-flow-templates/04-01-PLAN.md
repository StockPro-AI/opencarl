    Create `src/carl/setup.ts` with the following exports:

    1. `checkSetupNeeded(options: { cwd: string; homeDir: string }): SetupCheckResult`
       - Uses `findProjectCarl()` and `findGlobalCarl()` from paths.ts
       - Returns `{ needed: boolean; targetDir: string | null; reason: string }`
       - Priority: project `.carl/` first, fallback to `~/.carl/`

    2. `seedCarlTemplates(targetDir: string, templateDir: string): Promise<SeedResult>`
       - Uses recursive directory copy with idempotent skip logic
       - Skips existing files (never overwrites user content)
       - Returns `{ copied: string[]; skipped: string[]; failed: { file: string; error: string }[] }`
       - Uses `fs.promises` API with `recursive: true` for mkdir

    3. `buildSetupPrompt(): string`
       - Returns setup prompt message with `[carl]` prefix
       - Includes instructions: `/carl setup` or manual steps
       - Keep under 500 chars for visibility

    4. `runSetup(options: { cwd: string; homeDir: string; templateDir?: string }): Promise<SetupResult>`
       - Orchestrates check → seed flow
       - Reports progress with checkmark format: "Copying plugin... ✓"
       - On partial failure: continue what you can, report all failures at end
       - Uses `console.log` for verbose progress feedback

    Types:
    ```typescript
    interface SetupCheckResult {
      needed: boolean;
      targetDir: string | null;
      reason: string;
    }
    
    interface SeedResult {
      copied: string[];
      skipped: string[];
      failed: Array<{ file: string; error: string }>;
    }
    
    interface SetupResult {
      success: boolean;
      targetDir: string | null;
      seedResult?: SeedResult;
      error?: string;
    }
    ```

    - Do not prompt on every turn - check once and cache
    - Do not block plugin load on setup failure

       },
