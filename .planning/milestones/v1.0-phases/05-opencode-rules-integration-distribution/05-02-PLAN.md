Create tsconfig.build.json for building the OpenCode plugin distribution:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "CommonJS",
    "lib": ["ES2020"],
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "moduleResolution": "node"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
```

This configuration:
- Targets ES2020 for modern Node.js compatibility
- Outputs CommonJS (required for OpenCode plugin loading)
- Generates declaration files (.d.ts) for TypeScript users
- Excludes test files from build

<task type="auto">
  <name>task 3: Create build script and plugin entrypoint</name>
  <files>scripts/build-opencode.ts, src/plugin.ts</files>
  <action>
1. Create src/plugin.ts as the OpenCode plugin entrypoint:

```typescript
import { createCarlPluginHooks } from "./integration/plugin-hooks";

export default {
  hooks: createCarlPluginHooks(),
};
```

This file serves as the main entrypoint that OpenCode loads from dist/plugin.js.

2. Create scripts/build-opencode.ts build script:

```typescript
import { execSync } from "child_process";
import fs from "fs";
import path from "path";

const projectRoot = path.resolve(__dirname, "..");

console.log("[build] Compiling TypeScript...");
execSync("npx tsc --project tsconfig.build.json", {
  cwd: projectRoot,
  stdio: "inherit",
});

// Verify dist/plugin.js exists
const pluginPath = path.join(projectRoot, "dist", "plugin.js");
if (!fs.existsSync(pluginPath)) {
  console.error("[build] ERROR: dist/plugin.js not found after compilation");
  process.exit(1);
}

console.log("[build] OpenCode plugin built successfully");
console.log(`[build] Output: ${pluginPath}`);
```

3. Add npm scripts to package.json for OpenCode build workflow:
   - "build:opencode": "npx ts-node scripts/build-opencode.ts"
   - "prepublishOnly:opencode": "npm run build:opencode"

Note: The actual publish workflow uses package renaming strategy (research pattern 1).
