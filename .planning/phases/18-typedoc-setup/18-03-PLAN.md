# Current Issue
entryPointStrategy expand in typedoc.json creates a module for each .ts file. Categories only organize exports within modules, not at the navigation level. User sees command-parity, context-brackets, debug etc. instead of Configuration, Loading, Matching, etc.

# Solution
1. Create barrel file (index.ts) that re-exports everything from all modules
2. Switch to entryPointStrategy resolve with single entry point
3. TypeDoc will treat all exports as belonging to one module, making categories the primary navigation

# Files to re-export (from src/opencarl/):
- types.ts (Configuration, Matching, Signals types)
- loader.ts (Loading functions)
- matcher.ts (Matching functions)
- injector.ts (Injection functions)
- signal-store.ts (Signals functions)
- errors.ts (Errors)
- command-parity.ts
- context-brackets.ts
- debug.ts
- setup.ts
- validate.ts
- rule-cache.ts
- help-text.ts
- session-overrides.ts
- duplicate-detector.ts
</context>
